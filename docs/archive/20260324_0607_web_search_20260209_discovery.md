# web_search_20260209 — Discovery & Correction Log

**Date**: 2026-03-24 06:07 UTC
**Stack**: Bun + Hono + Mastra + Anthropic (`claude-sonnet-4-6`)

---

## 1. Initial Assumption (Wrong)

We initially believed `web_search_20260209` (Anthropic's latest web search tool with dynamic filtering) was **not available** through the Vercel AI SDK (`@ai-sdk/anthropic`), and therefore not usable in Mastra.

This was based on:
- The [AI SDK Anthropic docs page](https://ai-sdk.dev/providers/ai-sdk-providers/anthropic) only documenting `webSearch_20250305`
- No search results showing `webSearch_20260209` in the AI SDK
- Context7 docs returning only the older tool version

We built a raw `@anthropic-ai/sdk` script (`test-search-v2.ts`) as a workaround.

---

## 2. What Actually Happened

Digging into the `@ai-sdk/anthropic` CHANGELOG on GitHub revealed:

> **`@ai-sdk/anthropic@3.0.54`**: `feat(provider/anthropic): add support for Anthropic web tools web_fetch_20260209 and web_search_20260209`

Our installed version is `3.0.64` — the feature was already there.

**Verification**:
```
node_modules/@ai-sdk/anthropic/src/tool/web-search_20260209.ts  ← exists
node_modules/@ai-sdk/anthropic/src/anthropic-tools.ts           ← exports webSearch_20260209
```

The function `anthropic.tools.webSearch_20260209()` accepts the same config as `webSearch_20250305()`:
- `maxUses`
- `allowedDomains` / `blockedDomains`
- `userLocation` (type, city, region, country, timezone)

---

## 3. Root Cause of Confusion

The **code shipped before the docs were updated**. The AI SDK docs page still only shows `webSearch_20250305`. This is a common pattern with Vercel AI SDK — they maintain wrappers for ~15+ providers across multiple frameworks, and doc updates lag behind code releases.

**Lesson**: When checking feature availability, check the CHANGELOG and actual source (`node_modules`) before assuming something isn't supported.

---

## 4. Updated Implementation

The Mastra agent now uses `webSearch_20260209` directly:

```typescript
// src/mastra/agents/expert-search.ts
import { anthropic } from "@ai-sdk/anthropic";
import { Agent } from "@mastra/core/agent";

export const expertSearchAgent = new Agent({
  id: "expert-search-agent",
  name: "Expert Judiciaire Search Agent",
  instructions: `...`,
  model: anthropic("claude-sonnet-4-6"),
  tools: {
    web_search: anthropic.tools.webSearch_20260209({
      maxUses: 10,
      userLocation: {
        type: "approximate",
        country: "FR",
        region: "Île-de-France",
        city: "Paris",
        timezone: "Europe/Paris",
      },
    }),
  },
});
```

**No raw SDK workaround needed.** The full stack works:
`Mastra Agent → AI SDK → @ai-sdk/anthropic → webSearch_20260209 → Anthropic API`

## 5. Why Vercel AI SDK Lags Behind Native SDKs

| Factor | Detail |
|---|---|
| **Multi-provider abstraction** | Every new tool must fit into a provider-agnostic interface across OpenAI, Anthropic, Google, Mistral, etc. |
| **Framework matrix** | Tested across Next.js, Nuxt, SvelteKit, plain Node, edge runtimes |
| **Edge cases** | Provider tools in streaming, parallel tool calls, multi-turn citations, error serialization — all need handling |
| **Docs lag** | Code often ships 1–3 weeks before docs are updated |
| **Known issue** | `@ai-sdk/anthropic@3.x` had runtime incompatibility with `ai@5.x` (V3 vs V2 model interface mismatch) |

**Public roadmap**: No formal public roadmap exists. Feature tracking happens via [GitHub issues](https://github.com/vercel/ai/issues) and the [CHANGELOG](https://github.com/vercel/ai/blob/main/packages/anthropic/CHANGELOG.md).

## 6. Recommendation: Stay on Mastra + AI SDK

Now that `webSearch_20260209` works through the full stack, there is **no reason to drop down to the raw SDK** for this project. The Mastra + AI SDK path gives us:

- Dynamic filtering (web_search_20260209) ← **confirmed working**
- Automatic tool loop
- Structured output with Zod
- Multi-agent networks & memory
- Observability & evals
- Auto-exposed HTTP endpoints via Hono adapter
- Provider switching (swap to OpenAI/Gemini with one line change)

The raw `@anthropic-ai/sdk` script (`test-search-v2.ts`) is kept as a reference/fallback but is no longer the recommended path.
